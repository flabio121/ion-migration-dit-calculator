Method and Assumptions

Calculation chain

Transient current data -> baseline correction -> pulse/window selection -> ionic current component -> integrated ionic charge -> areal charge density -> mobile ion concentration -> conductivity -> mobility -> exportable report.

Implemented equations

  • Q_ion = integral[t1,t2] (I(t) - I_baseline(t)) dt
  • Q_A = Q_ion / A
  • F_bias = sqrt(1 + 16 V_bi / V_T) - sqrt(1 + 16 (V_bi - V_pre) / V_T)
  • N0 = 8 (Q_ion / (A F_bias))^2 / (q eps0 epsr V_T)
  • sigma_ion = L / (A R_ion)
  • mu_ion = sigma_ion / (q N0)
  • V_T = k_B T / q

Assumptions and limitations

  • The concentration expression preserves the validated Excel-model form currently implemented in the calculator.
  • Results depend strongly on baseline choice, integration window, active area, thermal voltage, permittivity, and ionic resistance.
  • The tool estimates effective mobile-ion characteristics; it does not uniquely identify a microscopic ionic species.
  • Initial electronic spikes should be excluded from the ionic integration window.
  • Severe shunts, unstable baselines, or incorrect units can invalidate the estimate.

Documentation and Troubleshooting

Input data

Use text, CSV, or TSV files with time/current columns. Files with PAIOS-style headers are supported by detecting the numeric data region after ### Data:.

Interpretation

Use the processed plot to verify that the shaded region contains the intended ionic response. Return-spike mode follows the current TDC workflow by integrating from the return-spike peak to the tail with a zero-current baseline.

Limitations

  • One dominant mobile ion approximation.
  • Results depend on identifying the ionic portion of the transient.
  • Active area and thickness must be accurate.
  • Ionic resistance must be measured or estimated independently.
  • Unstable baselines can invalidate the calculation.
